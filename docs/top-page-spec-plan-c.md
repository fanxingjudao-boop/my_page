# トップページ具体仕様（プランC / 本格プロダクト型）

## 0. 目的と設計方針
- 目的: 「見た瞬間にプロフェッショナル」と評価される、成果直結型トップページを実現する。
- 価値軸: **高級感ある体験 / 情報の明快さ / 信頼形成 / コンバージョン最適化 / 運用拡張性**。
- 方針:
  - 体験は動的に、情報設計は論理的に。
  - 全演出はパフォーマンスとアクセシビリティを優先。
  - 単一ページ開始（mainのみ）から、多ページ・多言語・CMS連携へ段階拡張可能に設計。

---

## 1. 全体情報設計（セクション構成）
1. グローバルナビゲーション（固定ヘッダー）
2. Hero（第一印象 + 主訴求）
3. Value Proposition（提供価値3〜4本柱）
4. Social Proof（実績・ロゴ・評価）
5. Featured Projects（代表実績 / 動的フィルタ）
6. Process（進行フロー）
7. Technology Stack（技術力可視化）
8. Pricing / Plan（目安プラン）
9. FAQ
10. Insight / News（CMS連携想定）
11. Contact CTA（強い行動導線）
12. Footer（信頼情報・法務・SNS）

---

## 2. セクション別の具体仕様

### 2.1 グローバルナビゲーション
**目的**
- 迷わせず、どのスクロール位置でも重要導線に到達可能にする。

**UI仕様**
- 上部固定ヘッダー（高さ72px、スクロール時60pxへ縮小）。
- 背景: 透明 → スクロール後に半透明グラス + 軽いぼかし。
- 左: ロゴ / 右: セクションリンク + Primary CTA「無料相談」。
- モバイル: ハンバーガーメニュー（オーバーレイ、フォーカストラップ対応）。

**動作仕様**
- 現在セクションのリンクをハイライト。
- スクロール方向でヘッダーの表示/非表示を制御（上スクロールで再表示）。

---

### 2.2 Hero
**目的**
- 5秒以内に「何が得られるか」を理解させ、行動を促す。

**UI仕様**
- 主見出し: 1文で成果訴求。
- 副見出し: 対象顧客・提供価値・差別化を簡潔化。
- CTA: 「事例を見る」「無料相談する」の2ボタン。
- 背景: 高品質な動的グラデーション/粒子演出（低負荷版あり）。
- 右カラム: 実績サマリーカード（例: 導入社数、継続率、平均改善率）。

**動作仕様**
- 見出しは文字単位でフェードアップ（過剰な演出は禁止）。
- CTAホバー時、軽い発光・奥行きを付与。
- 初回表示時のみアニメーション再生（再訪で軽量化）。

---

### 2.3 Value Proposition
**目的**
- サービス価値を短時間で理解させる。

**UI仕様**
- 3〜4カード構成: 速度 / 品質 / 運用 / 成果 など。
- 各カードにアイコン・短い説明・補足リンク。

**動作仕様**
- スクロール進入でstagger表示。
- カードホバーで境界線・影・アイコンが連動。

---

### 2.4 Social Proof
**目的**
- 信頼性の定量・定性証明。

**UI仕様**
- 導入企業ロゴ（グレースケール→ホバーでカラー）。
- 指標のカウントアップ（例: プロジェクト数、満足度）。
- 推薦コメント2〜3件（顔写真は任意）。

**動作仕様**
- 数値はIntersectionObserverで1回だけカウントアップ。

---

### 2.5 Featured Projects（動的）
**目的**
- 具体成果を示し、検討フェーズを前進させる。

**UI仕様**
- カード一覧 + カテゴリフィルタ（業界/課題/技術）。
- カード内容: サムネ・課題・施策・結果（数値）。
- 「詳細を見る」でモーダル開閉。

**動作仕様**
- フィルタ時はスムーズなレイアウト遷移。
- モーダルでBefore/After、採用技術、担当範囲を表示。

**データ設計**
- `data/projects.json` をデータソース化（将来CMS置換可能）。

---

### 2.6 Process
**目的**
- 依頼後の流れを透明化し、心理的不安を減らす。

**UI仕様**
- 4〜6ステップ（相談→提案→制作→改善）
- 各ステップに成果物例・所要目安を併記。

**動作仕様**
- タイムライン進行アニメーション（スクロール連動）。

---

### 2.7 Technology Stack
**目的**
- 技術的な信頼と対応範囲を明示。

**UI仕様**
- カテゴリ別（Frontend / Backend / Infra / Analytics）。
- 各ロゴに経験年数または適用実績を表示。

**動作仕様**
- フィルタで表示切替。
- モバイルはアコーディオンで圧縮表示。

---

### 2.8 Pricing / Plan
**目的**
- 検討の現実感を上げ、問い合わせハードルを下げる。

**UI仕様**
- 3プラン（Starter / Growth / Enterprise）
- 各プランに対象、内容、価格目安、納期目安。
- 「要件に応じて見積」導線を必須表示。

**動作仕様**
- 月額/一括切替トグル（必要に応じて）。

---

### 2.9 FAQ
**目的**
- 問い合わせ前の懸念を解消。

**UI仕様**
- 6〜10項目のアコーディオン。
- カテゴリタブ（契約/費用/進行/保守）。

**動作仕様**
- 開閉アニメーションは200ms程度、キーボード操作対応。

---

### 2.10 Insight / News（CMS連携想定）
**目的**
- 継続更新により専門性・鮮度を維持。

**UI仕様**
- 最新3件表示（タイトル、要約、公開日、タグ）。
- 記事一覧ページへの導線。

**データ仕様**
- 初期はローカルJSON、将来はHeadless CMS連携。

---

### 2.11 Contact CTA
**目的**
- 最短で行動に移してもらう。

**UI仕様**
- 強調背景 + 主要CTA + サブ導線（資料DL等）。
- フォームは別ページ遷移またはモーダル。

**フォーム仕様（推奨）**
- 入力: 会社名 / 担当者名 / メール / 相談内容 / 予算感（任意）。
- バリデーション: リアルタイム + 送信時。
- 状態: 送信中、成功、失敗を明示。

---

### 2.12 Footer
**目的**
- 信頼情報の集約と回遊補助。

**UI仕様**
- 会社情報、所在地、SNS、利用規約、プライバシー、特商法（必要時）。
- サイトマップリンク配置。

---

## 3. 非機能要件（プロ品質ライン）

### 3.1 パフォーマンス
- 目標:
  - LCP: 2.5s以内
  - CLS: 0.1以下
  - INP: 200ms以下
- 施策:
  - 画像最適化（AVIF/WebP + 遅延読み込み）
  - フォント最適化（subset / `font-display: swap`）
  - JS分割読み込み（重い演出は遅延実行）

### 3.2 アクセシビリティ
- WCAG 2.1 AA準拠を目標。
- コントラスト、フォーカス可視、ariaラベル、キーボード操作対応。
- `prefers-reduced-motion` 時はアニメーション抑制。

### 3.3 SEO
- title/description最適化。
- OGP / Twitter Card / 構造化データ（Organization, WebSite）。
- パンくず（多ページ化時）と内部リンク最適化。

### 3.4 計測・分析
- GA4 + Search Console導入。
- 主要イベント: CTAクリック、フォーム送信、スクロール到達率。
- ヒートマップは任意導入（過剰追跡は避ける）。

## 5. コンテンツガイド（品質担保）
- 見出しは「機能説明」でなく「成果訴求」を優先。
- 数値は可能な限り具体化（例: 改善率、期間、件数）。
- 専門用語は最小化し、必要時は補足を付与。
- トーン: 高級感 + 誠実 + 断定しすぎない信頼表現。

---

## 6. 受け入れ基準（Definition of Done）
- 初見ユーザーが10秒以内に提供価値を説明できる。
- 主要CTAが全デバイスで視認しやすい。
- Lighthouse（Mobile）でPerformance/SEO/Best Practicesが高水準。
- 主要操作がキーボードのみで完結可能。
- 実績セクションのフィルタとモーダルが破綻なく動作。

---

## 7. 次の実装タスク（この仕様からすぐ着手可能）

4. `main.js` でモジュール初期化
5. アニメーション実装（軽量優先）
6. Lighthouse計測と改善サイクル
